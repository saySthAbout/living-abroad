## 5. 비기능 요구사항

### 5.1 보안

| 요구사항 ID | 요구사항 |
|---|---|
| NFR-SEC-001 | 비밀번호는 평문으로 저장하지 않아야 한다. |
| NFR-SEC-002 | 보호 API에는 JWT 인증을 적용해야 한다. |
| NFR-SEC-003 | 분석 결과와 상담 이력은 소유자만 조회할 수 있어야 한다. |
| NFR-SEC-004 | DB 비밀번호, JWT Secret과 LLM API Key는 환경변수로 관리해야 한다. |
| NFR-SEC-005 | 민감한 개인정보를 일반 로그에 출력하지 않아야 한다. |
| NFR-SEC-006 | SQL Injection과 XSS에 대한 기본 방어를 적용해야 한다. |

### 5.2 성능

| 요구사항 ID | 요구사항 |
|---|---|
| NFR-PERF-001 | 일반 조회 API는 정상 환경에서 3초 이내 응답을 목표로 한다. |
| NFR-PERF-002 | AI 분석은 진행 상태를 사용자에게 표시해야 한다. |
| NFR-PERF-003 | 직업 설명 임베딩은 미리 생성하여 분석 요청 시간을 줄여야 한다. |
| NFR-PERF-004 | RAG 검색 결과는 상위 3~5개 청크로 제한해야 한다. |

### 5.3 신뢰성

| 요구사항 ID | 요구사항 |
|---|---|
| NFR-REL-001 | 동일한 모델과 입력값에서는 재현 가능한 결과를 제공해야 한다. |
| NFR-REL-002 | 모델 버전과 데이터 버전을 분석 결과에 기록해야 한다. |
| NFR-REL-003 | AI 서버 오류가 발생해도 분석 요청 상태를 확인할 수 있어야 한다. |
| NFR-REL-004 | RAG는 공식 출처가 없는 내용을 사실처럼 생성해서는 안 된다. |

### 5.4 사용성

| 요구사항 ID | 요구사항 |
|---|---|
| NFR-UX-001 | 분석 입력은 2단계 이하로 구성해야 한다. |
| NFR-UX-002 | 필수 입력값과 오류 항목을 명확히 표시해야 한다. |
| NFR-UX-003 | 분석 진행 중 사용자가 처리 중임을 인지할 수 있어야 한다. |
| NFR-UX-004 | 추천 결과에는 점수뿐 아니라 추천 이유를 함께 제공해야 한다. |
| NFR-UX-005 | 모바일과 데스크톱 화면에 대응하는 반응형 UI를 제공해야 한다. |

### 5.5 운영 및 유지보수

| 요구사항 ID | 요구사항 |
|---|---|
| NFR-OPS-001 | Frontend, Backend, AI Server와 DB를 분리하여 구성해야 한다. |
| NFR-OPS-002 | Docker 기반으로 동일한 실행 환경을 구성해야 한다. |
| NFR-OPS-003 | Backend와 AI Server 상태 확인 API를 제공해야 한다. |
| NFR-OPS-004 | 정책 문서에는 최종 검증일을 저장해야 한다. |
| NFR-OPS-005 | 버전은 `latest`가 아닌 확정된 버전으로 관리해야 한다. |

## 7. 시스템 요구사항

| 영역 | 요구사항 |
|---|---|
| Frontend | Vue 3 기반 SPA로 구현한다. |
| Backend | Spring Boot가 회원, 인증, 프로필, 분석 결과와 상담 이력을 관리한다. |
| AI Server | FastAPI가 비자 규칙, ML, 임베딩 분석과 RAG를 수행한다. |
| Database | PostgreSQL에 서비스 데이터를 저장한다. |
| Vector Search | pgvector에 정책 문서와 직업 임베딩을 저장한다. |
| API | Vue와 Spring Boot, Spring Boot와 FastAPI는 REST JSON으로 통신한다. |
| Deployment | Docker를 사용하여 각 서비스를 분리 실행한다. |

## 8. 완료 기준

프로젝트 MVP는 다음 조건을 모두 충족하면 완료된 것으로 본다.

1. 사용자가 회원가입과 로그인을 할 수 있다.
2. 사용자가 프로필과 분석 정보를 입력할 수 있다.
3. Vue에서 Spring Boot를 거쳐 FastAPI 분석을 요청할 수 있다.
4. 캐나다·호주·영국의 추천 결과가 생성된다.
5. 국가별 종합 점수와 세부 점수가 표시된다.
6. 경력과 국가별 직업군의 유사도가 표시된다.
7. 분석 결과가 DB에 저장되고 다시 조회된다.
8. AI 상담에서 공식 문서 기반 답변을 받을 수 있다.
9. AI 상담 답변에 출처와 기준일이 표시된다.
10. 다른 사용자의 분석 결과와 상담 이력에 접근할 수 없다.
11. 배포 환경에서 회원가입부터 결과 조회까지 전체 시연이 가능하다.
